What's my Deal?

I'm a platform, framework and code-agnostic programmer. Because of this, I have a unique perspective on compatability and portability.

Click here to see projects I have worked on. If you don't, you're going to be subjected to me talking about myself, below.

What's my Background?

I started coding in the eighth grade, before CSS was a thing.

When I was 17, I worked at Boston University's Engineering Information Technology department and became familiar with Bash shell scripting and networking. In the meanwhile, I learned C++ while attending BU.

Though Microsoft Visual Basic made me want to cry at times, this underpinning in C++ gave me a great foundation in object-oriented code that I've appreciated, to this day.

Later, I worked independently as an "emergency programming service", of sorts. If another programming team blew their deadline, they would call me.

Need to make that static page dynamic? Gotcha. Need to finish the filtering for those search results? Gotcha. Why do you have a post-mortem stack trace? Let's find out.

This exposure to different programming languages, CMSs, project management styles and teams is why I can make different operating systems, protocols and third-party applications play nicely with each other.

What's AUX?

In a nutshell: Systematic instruction on iterative design, prototyping, wireframing, user interviewing, usability testing, and front end development topics such as accessibility, performance budgeting and atomic code architecture

Prior to the Apprentice User Experience (AUX) program at Fresh Tilled Soil, I was working as a network engineer and freelance programmer. I wanted to focus on creating digital products in a human way and UX was a natural choice.

Specifically, I apprenticed as a front end developer to take a deep dive into HTML and CSS architecture. During the program, world-class mentors reinforced the skills to write clean, semantic code and craft consistently awesome user experiences. Later, this all culminated in participating in client work in different industry verticals.

So, what did I actually do?

AUX was structured as a complex set of problems and challenges, all solved and overcome by immersing oneself in evidence-based research and iterative design processes.

Highly competitive, entrance to AUX was gated by an initial case study on a previous project, which you can see here (it'll open in a new window), filtered by a "bootcamp" design sprint, which you can read about here, and converged at a presentation of our sprint to the other applicants and directors of the program.

Four people get in to the program each cycle.

What's my Process?

Here are a few things I know to work 100% of the time:

I Talk Before I Code

I don't rush into coding something that doesn't solve the problem.

It's not agile if it has to be done all over again.

I Focus on People

I'm different from other developers. I develop for humans.

People say, "content is king". Sadly, those people forgot about people.

I Code in Whatever

I do my best to master logic and code theory - everything else is easy

I have used both object-oriented and procedural programming languages and continue to learn as much as I can about both ends of the spectrum.

And When You're not Coding?

I like riding my bike, drawing in pen, gardening and obsessing over stationery.

Really, though, hanging out with my dog trumps all of those things.
